# Typography

Typography defines the visual style of text and plays a key role in creating clear, accessible, and consistent user interfaces. We use MUI's `Typography` component to apply predefined typographic styles that align with our brand guidelines and design tokens.

<Canvas>
  <Story name="Typography" of={TypographyStories.Typography} />
</Canvas>

## Usage

Use the `Typography` component whenever displaying text to ensure consistency across the application. Each variant maps to a semantic and stylistic purpose, allowing both accessibility and visual hierarchy.

<Source
  dark
  code={
    'import { Typography } from "@mui/material";\n\n<Typography variant="h1">Heading Text</Typography>'
  }
/>

## Guidelines

- Use semantic variants (`h1`–`h6`, `body1`, `caption`, etc.) to reflect the document structure.
- Avoid using raw `<h1>`–`<h6>` or `<p>` elements directly; always use the `Typography` component.
- Pair with appropriate spacing (`gutterBottom`, `margin`, `padding`) using MUI props or system styles.

## Custom Variants

In addition to standard MUI variants, we've defined custom ones to suit application needs:

- `appTitle`: Used in global headers or application-level branding.
- `pageTitle`: For top-level page titles.
- `tableTitle`: For table headers or data section labels.
